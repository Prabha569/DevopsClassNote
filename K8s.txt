🔹 PV (Persistent Volume) kya hota hai?
=======================================
PV = Cluster level storage
👉 Ye ek actual storage resource hota hai jo cluster me available hota hai

🔹 Problem kya thi? (PV kyu aaya)
==================================
By default:- Pod delete → data delete 😢

👉 Isko solve karne ke liye PV use hota hai
👉 Data pod ke bahar store hota hai

1. PV (Persistent Volume)
> Admin create karta hai
> Actual storage define karta hai

2. PVC (Persistent Volume Claim)
> User request karta hai storage

Jaise:- "Mujhe 1GB storage chahiye"

3. Pod
> PVC use karta hai
> Volume mount karta hai

🔹 PV ka flow (important for interview 🔥)
> Admin storage create karta hai → PV
> User request karta hai storage → PVC (Persistent Volume Claim)
> PVC bind ho jata hai PV se
> Pod PVC ko use karta hai

Flow:- PVC → PV → Actual Storage

🔹 PV Types:-
1. Static Provisioning
> Admin manually PV create kare

2. Dynamic Provisioning
> StorageClass use hota hai
> Auto PV create ho jata hai

🔥 Short Interview Answer (yaad karle bhai)
------------------------------------------

👉 “Persistent Volume Kubernetes ka cluster-level storage resource hai jo pod lifecycle se independent hota hai. Isko PVC ke through access kiya jata hai aur ye data persistence provide karta hai.”

English:-
=========
“Persistent Volume (PV) Kubernetes ka ek cluster-level storage resource hai jo pod ke lifecycle se independent hota hai. Iska matlab hai ki pod delete hone ke baad bhi data safe rehta hai. PV ko directly use nahi kiya jata, balki Persistent Volume Claim (PVC) ke through access kiya jata hai. Ye mainly stateful applications jaise databases aur file storage ke liye use hota hai jahan data persistence required hota hai.”
