✅ YAML Basic Rules & Syntax:-
-----------------------------
1. Key-Value Pairs

name: dhoondhu
age: 66

2. Indentation Matters (Always use (2 ya 4s)spaces, NOT tabs!)

student:
  name: dhoondhu
  age: 66

3. List (Array)
subjects:
  - math
  - science
  - history

4. List of Objects

students:
  - name: dhoondhu
    age: 66
    marks: 89
  - name: shyamu
    age: 65
    marks: 90

5. Comments
# This is a comment
name: dhoondhu  # inline comment

6. Multiline Strings
description: |
  This is a long text.
  It can go on multiple lines.
  Line breaks are preserved.

note: >
  This is another long text.
  Line breaks become spaces.

7. Booleans and nulls

pass: true
fail: false
unknown: null

8. Quoting Strings (optional unless string has special characters)

name: "dhoondhu"
message: 'Hello, world!'

Class--112/22
=============
=> Kubernetes mein Persistent Volume (PV) ek cluster-level resource hota hai, jisme aap storage ki details de dete ho (kaun sa backend, size, access mode, etc.).

=> PersistentVolumeClaim (PVC) ek user request hoti hai storage ke liye.

Binding tab hota hai jab:
------------------------
>•  "PVC ka requested storage, accessModes, and volume Mode match karta hai PV se."

> "Aur agar PVC ne kisi specific PV ka naam nahi diya ho (i.e. selector ya volumeName), to system automatically best match dhoondhta hai."

persistantVolume.yaml
====================
apiVersion: v1
kind: PersistentVolume 
metadata:
  name: pyaar-ka-storage
spec:
  storageClassName: gym-lover       
  volumeName: pyaar-ka-storage
  accessModes:
  - ReadWriteMany
  azureFile:
    secretName: shadikephotos-ka-creds
    shareName: honeymoon
  capacity:
    storage: 4Gi


pvc.yaml
=======
apiVersion: v1
kind: PersistentVolumeClaim 
metadata:
  name: honeymoon-photos
spec:
 storageClassName: gym-lover       
 volumeName: pyaar-ka-storage
resources:
  requests:
    storage: 1Gi 
volumeMode: Filesystem
accessModes:
- ReadWriteMany
